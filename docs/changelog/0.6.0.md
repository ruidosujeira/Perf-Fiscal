# Perf Fiscal 0.6.0

Released: 2026-06-05

## Highlights

- **Vue.js support** – Three new rules bring Perf Fiscal's performance discipline to Vue 3, covering both the Composition and Options APIs:
  - `vue-no-expensive-computed` – flags expensive or overly complex `computed` getters.
  - `vue-no-inefficient-watchers` – catches deep, nested, or derive-only watchers that should be `computed`.
  - `vue-optimize-reactivity` – guards against `reactive()`/`ref()` misuse and reactivity created inside loops.
  - New `vue` and `flat/vue` presets bundle the Vue rules alongside the framework-agnostic ones.
- **Rust core acceleration** – An optional Rust core now backs heavy analysis, with an incremental, content-hashed cache for near-instant re-runs and a `check-redos` guardrail that strengthens `no-redos-regex`. The plugin transparently falls back to its JavaScript implementations when the binary is absent.
- **Reliability fixes** – The Vue rules shipped broken in pre-release builds (grouped `:exit` selectors threw at runtime, and hand-rolled AST traversals recursed infinitely through `parent`). All three are now fully functional and covered by the regression suite.

## Upgrading

1. Update your dependency to `eslint-plugin-perf-fiscal@^0.6.0`.
2. Re-run your package manager install step.
3. For Vue projects, extend the Vue preset:
   ```js
   // Flat config
   import perfFiscal from 'eslint-plugin-perf-fiscal';
   export default [perfFiscal.configs['flat/vue']];
   ```
   ```jsonc
   // Classic config
   { "extends": ["plugin:perf-fiscal/vue"] }
   ```

React-focused projects keep using `perfFiscal.configs.recommended` / `plugin:perf-fiscal/recommended` with no changes.

## Rule Updates

- Added `vue-no-expensive-computed`, `vue-no-inefficient-watchers`, and `vue-optimize-reactivity`, each with a `strictness` option (`relaxed` | `balanced` | `strict`).
- Fixed grouped `:exit` selectors in the Vue loop-tracking rules — ESLint only strips a single trailing `:exit`, so the grouped form threw `Unknown class name: exit`.
- Fixed infinite recursion in the shared Vue AST traversals (`hasNestedReactivity`, `hasExpensiveOperations`) and rule-local walkers by skipping the cyclic `parent` reference.
- `vue-no-inefficient-watchers` now reports a single, most-specific diagnostic per watcher instead of stacking overlapping warnings.
- Added the missing `docs/rules/no-inline-context-value.md` and listed every Vue rule in both READMEs.

## Rust Core

- Incremental cache (`rust/perf-linter-core/src/cache.rs`): content-based hashing with a two-tier in-memory + disk cache for repeat runs.
- ReDoS bridge: `no-redos-regex` consults the Rust `check-redos` command when available, with a safe JavaScript fallback.
- See `docs/rust-core-improvements.md` for the full roadmap and current dependency status.

## Tooling

- Converted lazy `require()` of Node built-ins to top-level imports and documented intentional empty `catch` blocks — `npm run lint` is clean.
- Added a `prepublishOnly` guard so the published `dist` is always rebuilt and tested.
